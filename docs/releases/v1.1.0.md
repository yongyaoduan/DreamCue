# DreamCue v1.1.0

## Highlights

- Added Firebase sync for Android using Firebase Auth and Firestore.
- Added tenant isolation with remote documents scoped to `users/{uid}/memos`.
- Added a native macOS SwiftUI App with Firebase Auth REST login and Firestore REST sync polling.
- Tightened the macOS Settings layout and added memo-row accessibility labels for XCUITest coverage.
- Converted all user-facing UI text, prompts, tests, and documentation to English.
- Added Android Compose UI tests and macOS XCUITest coverage for core flows.

## Notes

- Firebase configuration is not committed. Fill Android string resources and `macos/DreamCueMac/FirebaseConfig.plist` before testing live sync.
- Android uses Firestore snapshot listeners for realtime updates.
- macOS uses periodic Firestore REST polling in this release.
- Firestore rules must restrict access to the authenticated user's own `users/{uid}/memos` path.
