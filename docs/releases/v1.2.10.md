# DreamCue v1.2.10

## Highlights

- Restores macOS sync after relaunch by storing the Firebase refresh token with the Keychain service `app.dreamcue.mac.firebase-sync`.
- Prevents Android `DreamCueViewModel.refresh()` from uploading local cues before the first Firebase Realtime Database snapshot has been merged.
- Keeps Android offline-created cues safe by uploading local state from the remote snapshot callback instead of from the initial screen refresh.
- Updates `DreamCueFirebaseOrderSyncTest` to grant `POST_NOTIFICATIONS` and write screenshots to the instrumentation `additionalTestOutputDir`.
