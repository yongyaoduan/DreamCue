# DreamCue v1.2.9

## Highlights

- Android now asks for notification permission on launch when the system requires it, so daily reminders are not silently blocked after a fresh install.
- Android daily reminders now use idle-aware scheduling instead of the regular inexact alarm path.
- DreamCue now shows a lightweight sign-in prompt on Android and macOS when the app opens without a sync account. The prompt can be skipped.
- Release notes are now read from the matching `docs/releases/<version>.md` file when GitHub publishes a tagged release.

## Verification

- Android unit tests cover notification permission policy and reminder scheduling policy.
- Android connected UI tests cover the startup sign-in prompt and the main screen flow.
- Android emulator testing confirmed the notification permission dialog, account prompt, scheduled reminder receiver, and visible DreamCue notification.
- macOS XCUITest covers the startup sign-in prompt and skip action.
- macOS live testing confirmed a scheduled DreamCue notification appears through the system notification banner.
- The release assets include the Android APK and macOS zip only.
